## What streamer proxy is for

Streamer proxy is the streamer-side service and security boundary.

It owns:

- Twitch OAuth
- Twitch access token
- Twitch refresh token
- Twitch numeric IDs
- Twitch chatters endpoint
- Dota lobby status endpoint
- invite endpoint
- future Steam/Dota Game Coordinator connection
- Steam credentials in local `.env`

It must not own:

- web dashboard UI
- player database
- queue ranking logic
- backend session/auth UI

## Current active problem

Twitch sends the user back to:

```text
https://test.raze1x6.mom/twitch/callback?code=...
```

This means Twitch OAuth authorization succeeds and a temporary code is issued.

However, `TWITCH_ACCESS_TOKEN` and `TWITCH_REFRESH_TOKEN` are still empty in streamer proxy `.env`:

```env
TWITCH_ACCESS_TOKEN=
TWITCH_REFRESH_TOKEN=
```

The latest logs showed no clear `/twitch/callback` hit reaching uvicorn after browser redirect.

Current likely causes:

1. Nginx route for `https://test.raze1x6.mom/twitch/callback` is not proxying to streamer proxy.
2. Callback route exists but is not visible in logs due to nginx/domain routing.
3. Callback route exists but token exchange/saving is incomplete.
4. Callback route may require API key incorrectly; Twitch will not send `X-Api-Key`.

## First checks for current Twitch callback problem

Run on streamer server:

```bash
grep -R "twitch/callback\|callback" -n /opt/dota-lobby-streamer-proxy/app
curl -i "http://127.0.0.1:8081/twitch/callback?code=test"
journalctl -u streamer-proxy -n 80 --no-pager
```

Interpretation:

- `404 Not Found`: callback route is missing or has a different path.
- `403 Unauthorized`: callback is incorrectly protected by API key.
- `500 Internal Server Error`: callback exists but token exchange/saving failed.
- No log entry when visiting public URL: nginx is not routing callback to streamer proxy.

If local callback works but public callback does not, inspect nginx:

```bash
nginx -T 2>/dev/null | grep -n "test.raze1x6.mom\|8081\|twitch"
```

The public domain must proxy to:

```text
http://127.0.0.1:8081
```

for `/twitch/callback`.

## Rules for future AI work

Always update these files when changing the project:

```text
docs/CURRENT_STATUS.md
docs/AI_NOTES.md
docs/project_context.md
PROJECT_FILES.txt
```

Do not rely only on chat history.

Before changing code, check:

1. Which repo is responsible: backend or streamer proxy?
2. Current `git status`.
3. Relevant service logs.
4. Whether `.env` values are local-only and not committed.
5. Whether `PROJECT_FILES.txt` must be regenerated.
6. Whether a change affects public installation/setup instructions.

When adding/removing/moving files, regenerate:

```bash
find . -type f \
  -not -path './.git/*' \
  -not -path './.venv/*' \
  -not -path './venv/*' \
  -not -path './__pycache__/*' \
  -not -name '.env' \
  -not -name '*.pyc' \
  | sort > PROJECT_FILES.txt
```
