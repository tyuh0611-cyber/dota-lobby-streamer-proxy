# Project context — streamer proxy

## Main strategy

Streamer proxy isolates all streamer-sensitive integrations from the backend.

Backend should never directly touch Twitch tokens, Steam credentials, or Dota Game Coordinator.

## Architecture

```text
Backend Control Center
  -> HTTP request with X-Api-Key
  -> Streamer Proxy
  -> Twitch / Steam / Dota
```

## Streamer proxy responsibilities

- Generate Twitch OAuth auth URL
- Receive Twitch OAuth callback
- Store Twitch access/refresh tokens locally
- Get Twitch chatters
- Store/resolve Twitch numeric IDs
- Expose limited endpoints to backend
- Later run Steam/Dota GC adapter

## Backend responsibilities

Backend only calls streamer proxy and receives safe responses such as lobby status, chatters, invite result.

## Required setup inputs

During install/setup, streamer should provide:

```env
PUBLIC_BASE_URL=https://test.raze1x6.mom
PROXY_API_KEY=<long random shared API key>
TWITCH_CLIENT_ID=<from Twitch Developer Console>
TWITCH_CLIENT_SECRET=<from Twitch Developer Console>
DOTA_MOCK_MODE=true
```

Later for real Dota:

```env
STEAM_USERNAME=
STEAM_PASSWORD=
STEAM_SHARED_SECRET=
DOTA_MOCK_MODE=false
```

## Values streamer should not manually provide

These should be obtained automatically by OAuth / Twitch API:

```env
TWITCH_ACCESS_TOKEN=
TWITCH_REFRESH_TOKEN=
TWITCH_BROADCASTER_ID=
TWITCH_MODERATOR_ID=
```

## Twitch OAuth redirect

The OAuth redirect URI generated by streamer proxy must use `PUBLIC_BASE_URL`.

For current test domain:

```text
https://test.raze1x6.mom/twitch/callback
```

Twitch Developer Console must have the exact same redirect URL.

Common failure:

```text
redirect_mismatch
```

Cause:

```text
redirect_uri=http://localhost:8081/twitch/callback
```

Fix:

```env
PUBLIC_BASE_URL=https://test.raze1x6.mom
```

and code must generate:

```text
{PUBLIC_BASE_URL}/twitch/callback
```
